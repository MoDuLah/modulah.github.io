What is Pythagoras CIS?

Pythagoras CIS is a Tampermonkey userscript for Torn company management. It adds a local company workspace for the training ledger, FIFO train planner, staff and director history, Sunday report analytics, licensing, backups, and themed reports.

How do I install it?

Install a userscript manager such as Tampermonkey or use Torn PDA userscript support, then install the built public script file from your release link.

After installation, open Torn and use the Pythagoras CIS panel to set your API key, company ID, theme, and support options.

What API key access does it need?

The script uses a Torn API key only when you press a sync button. It does not run background polling or automated request chains.

If you do not tick Remember locally, the key is not saved by the script when settings are stored.

Where is my data stored?

Ledger entries, staff history, analytics, settings, and license data are stored locally in Tampermonkey storage, with a browser storage fallback.

Nothing is sent to a Pythagoras Project server by the userscript. Exports are files you choose to create and share.

How does the Training Ledger calculate costs?

The ledger starts from your price per train, then applies the configured discount rules:

  • Manual discount
  • Merit discount, defaulting to 1 percent per merit up to 10
  • Loyalty tiers such as 10:5,25:7,50:10
  • Global promo discounts such as 25 percent or 50 percent

Totals recalculate as you edit rows, including paid, unpaid, done, undone, used trains, and remaining trains.

How does the Train Planner work?

The planner uses FIFO order for paid train entries. It gives paid users up to the configured daily cap, with a maximum of 8 paid trains per day, then fills remaining capacity with sponsored rotation users.

Daily supply is based on company stars, from 1 to 10, with one extra train when the Trainer role is assigned in settings.

What does the Timeline track?

The timeline normalizes company news entries and classifies them into categories such as Hire, Application, Left, Fired, Director, Rating, Sunday report, Training, Withdraw, Deposit, Wage, Funds, and Other.

Timeline events are also used to build current and past staff/director lists where the news text gives enough detail.

How are weekly analytics calculated?

When Sunday report entries include customers and income, CIS parses them into weekly rows. It shows week number, date range, customers, income, profit per customer, comparison against the previous week, and comparison against the all-time average.

If a Sunday report has unusual wording, it may need a parser update before all values are detected.

What is the wage system?

The wage helper estimates staff wages from MAN, INT, END, merits, addiction penalty, and inactivity penalty settings. It is a configurable calculator, not an official Torn payroll value.

Use it as a weekly planning aid and adjust the weights to match your company policy.

What do Supporter, Company Boost, and Ultimate unlock?

The free version covers core features. Supporter unlocks premium features such as full data backup and supporter extras. Company Boost is aimed at advanced company tools. Ultimate includes the full bundle and premium support.

Licenses can be 30-day or lifetime. Trials are 7 days per type.

What analytics benefits does each tier include?

Each tier is aimed at a different management workload. Free gives the core local tools, while paid tiers add more ways to review company health, compare decisions, and move data into reports.

Supporter

  • Full data export and import for ledger, staff history, analytics, settings, and license backup.
  • Newsletter and report controls that make weekly summaries easier to share with staff.
  • Supporter badge snippet for forum signatures and supporter visibility.

Company Boost

  • Advanced company tools for directors who monitor stock, company profile, detailed stats, and staff effectiveness.
  • Better operational context for training demand, role coverage, stock warnings, and company trend checks.
  • Company-bound licensing so the boost follows the intended company workspace.

Ultimate

  • Everything from Supporter and Company Boost in one bundle.
  • Best fit for directors who want backups, reports, advanced company monitoring, badges, and premium support together.
  • Useful when the same person manages both staff planning and longer-term company analytics.
Why is my key rejected?

Common causes are a mismatched Torn user ID, a mismatched company ID for boost keys, an expired timed key, or a damaged copied key.

Use Settings -> Import license key, paste the full key, then check the message shown by the script.

What can I export?

Supporter or higher can export and import full data, including ledger, staff history, analytics, settings, and license data.

Anyone can create a themed HTML report, newsletter-ready HTML, and forum-ready snippets where those features are available in the installed build.

Does popup mode persist?

Yes. If you switch to popup mode, CIS remembers that mode across Torn page changes. If the browser blocks automatic popup restoration, the embedded page shows a small badge so you can reopen it with one click.

Can I change the colours and theme?

Yes. Settings includes theme presets and colour pickers for role and contract colours. You can choose a preset such as MoDuL dark or Blue and gold, then adjust role colours with HEX fields or colour pickers.

The panel is not showing. What should I check?
  • Confirm the userscript is enabled in Tampermonkey or Torn PDA.
  • Reload Torn after installing or updating the script.
  • Check whether the UI is minimized into the small badge.
  • If popup mode is enabled, check whether your browser blocked or hid the popup window.
The API sync failed. What should I check?
  • Confirm your API key is entered in Settings.
  • Confirm the company ID is correct for company syncs.
  • Wait and retry if Torn API is unavailable or rate limited.
  • Use one sync button at a time; CIS is designed around one user action and one request.
How do I report a bug?

Use Settings -> Report bug inside the userscript. CIS copies an auto-filled diagnostic block with version, page URL, user ID, company ID, ledger count, timeline count, and timestamp.

Paste that diagnostic block into your support message so the issue can be reproduced faster.

How do I contact support?

Use the configured Contact support link in Settings. If the link is missing, ask the script maintainer for the current support page or install an updated public build with support constants included.

Need a parser fix? Save the raw timeline line, the category you expected, and the category CIS selected. That is usually enough to patch the classifier cleanly.
