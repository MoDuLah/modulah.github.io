# Pythagoras Project - CIS Changelog

## v2.9.8 - Persistence and operations polish

- Wage-role and stock-management edits now persist immediately, with company-scoped local recovery for stock settings and GM-backed UI collapse preferences.
- Timeline news is re-deduplicated during migration/cache restore, including exact legacy copies that carried different stored IDs.
- Operating Balance defaults to Daily graphs, remembers the selected scale, repairs bracketed zero ad-budget history, and uploads dated wage/ad-budget totals with daily reports and business syncs.
- Staff now owns the Efficiency column and stat-based suggested-role analysis; the duplicate employee-effectiveness table was removed from Business Profile.
- Training Schedule now uses current Profile train availability up to the fixed 20 cap, allows repeated candidates when slots exceed the roster, and follows the configured Queue standard.
- Training Schedule, Settings, notification rules, discount rules, training-order scrolling/sorting, fixed notifications, and equal-height Timeline cards received the requested layout pass.
- Training-order Paid and Done states are normalized before database upload so derived on-screen status and stored status cannot diverge.

## v2.9.3 - Training news grouping fix

- Grouped timeline training rows now collapse named employees only by their own identity, not by same-minute role/count fallback buckets.
- Same-timestamp one-train events for different employees now stay as separate grouped rows instead of being summed under the first matching role.
- Torn company-news event IDs are now the source of truth for training dedupe; employee/minute/count matching is only used for legacy or generated rows with no source event ID.

## v2.9.2 - Balance sync stability

- Advertising budget history now records Torn API V2 profile values as explicit dated rows and ignores older unmarked zero rows in Balance.
- The Pythagoras API now returns ad-budget raw metadata and rejects unmarked zero ad-budget rows, while still allowing future explicit zeroes from updated clients.
- Balance wages now use current synced employee wages and saved wage history only; wage estimates are no longer included in Balance profit.
- Notification hide/dismiss choices now persist locally until the next 18:10 TCT business reset, even after a page refresh.
- Weekly operating performance graphs now use adaptive Y-axis tick spacing so weekly labels do not overcrowd.
- Services Sold now explains that Torn's stock endpoint does not specify whether `sold_amount` is daily, weekly, monthly, or lifetime.
- The title bar now shows the userscript version and compact Sync buttons beside it.

## v2.9.1 - Stock settings persistence

- Stock card restock toggles, warning thresholds, warning modes, and restock quantities now save to a dedicated stock settings map.
- Stock settings now update the 24-hour sync cache immediately on input/change, so fast Torn page switches and refreshes keep the latest values.
- Stock sync, suggested restock, stock import/export, and legacy migration now preserve stock settings instead of falling back to default row values.

## v2.9.0 - API DNS Update

- New API DNS location for improved security and SSL compliance.
- The active CIS API host is `https://pp-api.sokin.xyz`.

## v2.8.12 - Advertising budget history

- Business/Profile sync now records Torn API V2 `advertisement_budget` as dated advertising-budget history.
- Balance now uses the latest observed advertising budget on or before each report date instead of leaving every ad-budget row blank.
- The Pythagoras API now stores advertising-budget history separately from the latest company settings so older dated values are preserved for statistics.

## v2.8.11 - Risk strike persistence

- Visible risk notifications now record same-day risk-strike history, so warnings cannot appear without also updating the member record.
- Staff sync snapshots now include `strikeHistory` and raw strike-history data for the Pythagoras API snapshot path.
- Risk-strike updates are also saved as staff-card edits and queued for staff/workspace cloud sync.

## v2.8.10 - Staff load guardrail and Mechanic Shop training rotation

- Cloud/database member loading now treats a valid left date on or after hire date as past staff before the current staff list is built.
- Mechanic Shop auto training now ranks free/sponsored slots by lowest known total working stats first, with a large projected-training penalty so Lucas-style repeats cannot dominate the ninth slot.
- Confirmed the active Balance ledger markup no longer contains the unreliable Employees column.

## v2.8.9 - Staff departure guard and Balance cleanup

- Company news left/fired events now override stale current-staff snapshots so departed employees cannot reappear from cached, cloud, profile, or employee-sync data unless a newer hire event exists.
- Employee sync now runs full staff dedupe after merging API rows, not only director filtering.
- Balance ledger and generated balance reports no longer show the unreliable Employees column.

## v2.8.8 - Business Profile and Balance render fix

- Business Profile daily/weekly fallback now reads lightweight report rows instead of building the full Balance ledger during profile render.
- Balance ledger now caches staff hire/left and wage-history data once per render instead of recalculating it for every report date.
- Balance wage lookup uses saved previous/new wage history when available and avoids repeated expensive staff date inference during table and graph rendering.

## v2.8.7 - Loading, rate limits, and Balance fixes

- Torn API calls now run through a shared throttle queue at roughly 27 calls per minute to avoid account-wide rate-limit bursts.
- Startup now renders the panel before rebuilding stored timeline analytics so loading does not appear frozen on large histories.
- Business Profile daily and weekly income/customer rows now fall back to synced daily report data when direct profile fields are empty.
- Userscript loading is scoped to `companies.php` and now runs at `document-end` for faster Torn page loading.
- Balance ledger no longer stamps current ad budget, rating, wages, or staff count onto every historical row; unknown ad budget/rating values stay blank.

## v2.8.6 - Training priority and Balance graphs

- Sponsored/free training slots now prioritize eligible staff with the largest role-stat gaps, falling back to the lowest total working stats when no role requirement is configured.
- Projected sponsored picks now reduce that member's immediate priority so the same ninth slot is less likely to repeat across planned days.
- Daily Balance was renamed to Operating Balance/Balance and the performance graph now includes Wages as a selectable money line.

## v2.8.5 - Torn Company API V2 sync

- Sync center Business, Company details, Employees, Stock, and Company news now use Torn API V2 Company endpoints.
- V2 company profile data now feeds both Business profile and Company details, including extended fields such as funds, trains, upgrades, storage capacity, and company value when the key allows it.
- Employee sync now understands V2 position, working stats, effectiveness, wage, joined date, status, and last action shapes.
- Company age display now rejects invalid tiny founded timestamps and falls back to the company age day count.
- Business Profile now shows founded date from the known company start estimate and staff as full company headcount.

## v2.8.4 - Database bootstrap repair

- Business profile loading now merges saved database company profile and company detail records into the active workspace during CIS bootstrap.
- Company funds, popularity, efficiency, environment, storage, value, staff capacity, daily figures, and weekly figures now load from the backend instead of rendering empty defaults after page load.
- Company identity, rating, director, founded date, and sync timestamps are still refreshed from the current backend company row.

## v2.8.0 - Cloud workspace sync hardening

- Added server-backed sync paths for staff-card edits, settings, training orders, and full company workspace mirrors.
- Company workspaces are keyed by the active user and company so separate companies can keep separate saved data.
- Cloud sync failures now keep local edits available and show a clear toast instead of silently discarding the change.

## v2.7.0 - Company workspace workflow

- Added the Data workspace controls for loading, saving, exporting, importing, and clearing active company data.
- Added saved company workspace handling so directors can reconnect to the correct cloud-backed company state after refresh.
- Added clearer privacy copy around local profile storage and backend verification.

## v2.6.5 - Script-side cleanup

- Removed obsolete client-side account UI, badge snippets, old saved-key storage, and local unlock prompts from the userscript.
- Removed local feature gates and stale disabled-control messages. The userscript now keeps UI, sync helpers, display formatting, and local calculations.
- Removed old local 7-day retention trimming and pass-through access helpers from the script-side state path.
- Removed dead handlers and report options tied to the old activation panel.

## v2.6.4 - Compliance cleanup

- Removed the wildcard userscript connect permission and limited external requests to Torn API and the Pythagoras API.
- Replaced local key-protection string helper functions with plain constants so the source does not look obfuscated during review.
- Removed lifetime wording and local signed lifetime-key acceptance from the userscript payload validator.
- Updated About/support wording to refer to in-game supporter notes and Torn-dollar/item contributions.

## v2.6.3 - Server-side premium enforcement

- Premium report and export buttons now request server-generated output from the Pythagoras API instead of generating premium-value output locally.
- Added backend endpoints for server-limited history queries, advanced weekly analytics, report generation, JSON exports, and planner generation.
- Added endpoint-level tier checks so edited client buttons cannot bypass Supporter, Company Boost, or Ultimate requirements.
- Updated the tier gate migration so Supporter keeps history/report benefits while advanced analytics and planner generation remain Company Boost/Ultimate server features.

## v2.6.2 - Local identity and entitlement refresh

- Fixed remembered API key handling so the local profile keeps the explicit Remember locally choice instead of deriving it from whether the key field is currently populated.
- Preserved local identity fields when cloud bootstrap data is loaded, preventing older cloud settings from blanking the API key or company/user IDs.
- After a successful Torn key check, CIS now immediately reloads the cloud workspace and entitlement so the active tier is applied without a page refresh.
- Lifetime-style backend entitlements no longer require a timed expiry field when the signed backend response marks the entitlement as active.

## v2.6.0 - Cloud workspace architecture

- Added the server-backed workspace migration path. The userscript now writes only the local profile fields: API key, user ID, username, and company ID.
- Added bootstrap loading from the CIS API and one-time legacy local data migration to the cloud workspace normalizer.
- Removed lifetime licensing from the current issue flow; CIS plans are now 1m, 3m, 6m, or 12m.
- Added server-side entitlement support for Free 7-day visibility, Supporter, Company Boost, and Ultimate.

## v2.5.9 - V2-only local storage
