Landlord Tenant Ledger

A Torn property ledger for landlords and renters that fills leases from property data, shows subscriber status, keeps partner homes tidy, and asks the server for ROI/listing suggestions.

At a glance

  • Fetch owned, current, and partner properties into one readable lease list
  • Merge shared homes and duplicate property rows by Torn property ID
  • Scan ROI/listings through the hosted server without exposing raw market pages

Description

Landlord Tenant Ledger is built for Torn players managing rented homes, partner homes, contract dates, and rental status. The current client keeps the visible workflow simple: Setup, Fetch, Leases, and Suggestions.

The old visible price scanner has been removed. The browser client checks paid access, fills ledger rows, reloads Torn contract details, and asks the hosted server for market summaries and sale/rent suggestions.

Supporter Access

Install the userscript, save a public Torn API key in Setup, then press Check subscription. Setup shows whether the Torn user from that key is a subscriber.

Until the ledger backend route is live, property fetching can fall back to direct Torn property calls so the ledger can still fill owned homes, current homes, and partner contracts.

What this tool does

Leases

Shows home type, owner, renter, rent total, rent per day, start/end dates, days left, status, notes, and a status filter.

Fetch

Pulls the user's owned homes, current home, and partner properties/contracts when that data is available.

Suggestions

Displays server-returned ROI/listing suggestions for the user's own properties, including sale/rent and duration guidance.

Backups

Exports the local ledger as JSON or CSV for recovery and spreadsheet checks.

Backend Work

The client expects /api/landlord-ledger/sync on the hosted service. That endpoint should verify paid entitlement, fetch property data, merge duplicate homes, calculate suggestions, and paginate market scans server-side. The Scan ROI/Listings button sends scope: "scan-owned-market".

The exact response shape and pagination notes are documented in SERVER_API_CONTRACT.md.
